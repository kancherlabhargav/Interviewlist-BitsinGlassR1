Interview Debrief

What They Asked Me in a Java Full Stack Interview

A complete walkthrough of every topic that came up — from Streams to Security, SQL to Spring Boot internals.

The interview wasn't just about getting the right answer — it tested how well you connect the dots between code, architecture, and real-world decisions.

01

Java 8 Streams — Think in Pipelines

They gave a list of transactions and asked to group them by type and sum up the total amount per type. Classic Collectors.groupingBy() with Collectors.summingDouble(). They're testing whether you think declaratively — not just loop-and-accumulate.

Stream API Collectors groupingBy Functional Java
02

SQL — The Query That Never Gets Old

Write a query to find total quantity sold per product in the last 30 days. Straightforward — GROUP BY product_id with a WHERE order_date >= NOW() - INTERVAL 30 DAY. But they're really watching how you handle date filtering, aggregation, and whether you mention indexing for performance.

GROUP BY Date Filtering Aggregation Indexing
03

JPA — Same Query, Application Layer

They took the same SQL question and flipped it: how would you do this from the app? Expecting you to compare Spring Data JPA derived queries, @Query with JPQL, native queries, and the Criteria API — and know when each one makes sense.

Spring Data JPA JPQL Criteria API Repository Pattern
04

REST API Design — Beyond Just CRUD

Build /employees/{id} — simple, right? But they wanted proper HTTP semantics: 200 for success, 404 when not found, 500 for server errors. Then they pushed into @ControllerAdvice for global exception handling. They want to see you design APIs that behave correctly, not just return data.

@RestController ResponseEntity @ControllerAdvice HTTP Status Codes
05

Secure Coding — The SQL Injection Conversation

How do you prevent SQL injection? Separate query structure from input — parameterized queries, prepared statements, never concatenating user input. They also asked about real vulnerabilities you've handled and how you identified them. They want proof you think about security in practice, not theory.

PreparedStatement Parameterized Queries OWASP Vulnerability Management
06

Java Fundamentals — Back to Roots

What's a Predicate in Java 8? It's a functional interface that takes input and returns boolean — the building block of filtering in Streams. Then came OOP principles — encapsulation, inheritance, polymorphism, abstraction. Sounds basic, but explaining polymorphism with a real example separates good from great.

Predicate<T> Functional Interfaces OOP Polymorphism
07

Spring Boot Internals — What's Really Happening

How does Spring Boot differ from traditional Spring? Auto-configuration, embedded servers, opinionated defaults, starter POMs. Then the deep cut: how does auto-configuration actually work? — @EnableAutoConfiguration, spring.factories / AutoConfiguration.imports, @Conditional annotations. This is where you show you've looked under the hood.

Auto-configuration @Conditional Starter POMs Embedded Tomcat
08

Architecture — Microservices Under the Microscope

How do microservices communicate? REST, gRPC, message queues — and when to use which. Then the challenge: is your project actually microservices, or just a distributed monolith? They also probed design patterns used — Saga, Circuit Breaker, API Gateway. This tests architectural honesty.

Service Communication Design Patterns Distributed Monolith Architectural Honesty
09

Performance Optimization — Make It Fast

What optimization techniques did you use in your last project? They want concrete answers — query optimization with execution plans, connection pooling (HikariCP), caching strategies (Redis, Spring Cache), lazy vs eager loading, database indexing, N+1 query detection. Real war stories win here.

Query Tuning Caching HikariCP N+1 Problem
10

DevOps Awareness — Ship It

CI/CD involvement — do you just push code and pray, or do you understand the pipeline? Jenkins, GitHub Actions, Azure DevOps. Then Kubernetes: do you know pods, services, deployments, config maps? They're not expecting a DevOps engineer, but they want someone who isn't blind past git push.

CI/CD Kubernetes Docker Pipeline Awareness
11

UI-Backend Integration — Bridging the Gap

How do you call APIs from React? Axios vs fetch, async/await, error handling, loading states. What's your role in frontend-backend integration — do you just hand off a Swagger doc, or do you sit with the frontend team and debug CORS issues together? They want a full-stack mindset, not just a backend dev who tolerates React.

React + Spring Boot Axios / Fetch CORS API Contract
12

Security Deep Dive — Lock It Down

Authentication vs Authorization — who are you vs what can you do. Then the stack: JWT, Basic Auth, OAuth — and when each fits. How does JWT validation work in Spring Boot? Filters. Which led to: Filters vs Interceptors — Filters are servlet-level (before Spring), Interceptors are Spring MVC-level (after DispatcherServlet). This is where depth matters.

JWT OAuth 2.0 Spring Security Filters vs Interceptors
